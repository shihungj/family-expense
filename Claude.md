# 家庭每月支出明細系統 CLAUDE.md

## 專案架構
- 前端：HTML + CSS + Vanilla JS，單一檔案 index.html
- 後端：Google Apps Script（Code.gs）
- 資料庫：Google Sheets
- 部署：GitHub Pages（https://shihungj.github.io/family-expense/）

## Google Sheets 工作表結構

### Transactions
欄位順序：入帳日、消費日、銀行、消費明細、金額、歸屬、帳單月份、批次編號
- 入帳日、消費日、帳單月份格式：YYYY/MM/DD 或 YYYY/MM（純文字）
- 金額：數字，讀取後必須 parseFloat() 處理
- 歸屬允許值：世鴻應付、慧鳳應付、共同支付、尚未分類

### BankSettings
欄位：銀行、結帳日、銀行縮寫
- 玉山銀行、13、ESUN
- 中國信託、25、CTBC
- 富邦銀行、26、FUBO
- 永豐銀行、26、SINO
- 聯邦銀行、12、UNIO

### MerchantRules
欄位：關鍵字、歸屬
- 新增規則插入第二列（標題列下方），不附加到最後
- 所有關鍵字比對不分大小寫
- 關鍵字比對包含原則（消費明細包含該關鍵字）

### SystemSettings
欄位：key、value
- 儲存帳號密碼，密碼以 Base64 編碼

## CSV 匯入格式
欄位：入帳日、消費日、銀行、消費明細、金額
- 第一列為標題列，固定跳過
- 空白列忽略，只處理有資料的列
- 帳單月份由使用者手動選擇（下拉選單，可選範圍當月往前 6 個月）
- 選擇超過上個月時跳橘色警示（可繼續）

## 批次編號規則
格式：銀行縮寫-YYYYMM，例如 ESUN-202605
- 系統於匯入時自動產生
- 手動新增交易時也自動產生
- 匯入前檢查重複，重複則阻擋並提示

## KPI 計算規則
- 世鴻應付 ＝ 世鴻應付總額 ＋ 共同支付總額 ÷ 2（餘數歸世鴻，無條件進位）
- 慧鳳應付 ＝ 慧鳳應付總額 ＋ 共同支付總額 ÷ 2（無條件捨去）
- 共同支付 ＝ 共同支付原始總額
- 本期總支出 ＝ 全部交易金額總和

## 支付工具關鍵字（消費明細欄位，不分大小寫）
- LINE Pay：包含「連加」或「連支」
- iCASH Pay：包含「icash pay」
- 悠遊卡：包含「悠遊卡」
- 悠遊付：包含「悠遊付」
- 車麻吉：包含「中油條碼_autopass」
- 信用卡：以上五個關鍵字都不符合

## 安全機制
- 前端 sessionStorage 儲存登入 Token
- 每頁載入檢查 Token，不存在跳回登入頁
- 登入後統一進入首頁
- 登出後重新登入：一般登出回首頁，密碼變更登出回變更密碼頁
- Apps Script 每次請求需帶驗證碼：family-expense-2026（Base64 編碼）

## 開發規則
- 金額欄位讀取後一律 parseFloat()
- 全形字轉半形：使用 `toHalfWidth()` 函式
- 前端與後端分開修改，每次只改一個檔案
- Code.gs 修改後必須重新部署 Apps Script
- index.html 修改後推送 GitHub
- 每次完成後自動執行 git add . / commit / push

## 帳單月份選擇器規則（首頁）
- 頂部顯示 `< 2026-05 >` 格式
- 點擊年月數字跳出選擇器，可快速跳轉
- 左右箭頭逐月切換
- 預設顯示當前月份
- 其他頁面移除月份選擇器

## 全部交易明細頁規格
- 表格欄位順序：勾選、項次、帳單月份、銀行、消費日、消費明細、金額、歸屬、操作
- 表格固定高度 600px，內部捲動軸
- 標題列固定不隨捲動
- 篩選列在標題列上方，只保留外框線

**篩選與排序規則**
- 銀行：下拉篩選 ＋ 排序
- 消費明細：關鍵字篩選
- 金額：排序
- 歸屬：下拉篩選
- 帳單月份：下拉篩選 ＋ 排序
- 其他欄：無篩選無排序

**多選與批次操作**
- 每列勾選框，標題列勾選框可全選
- 勾選後顯示批次操作：修改歸屬＋刪除
- 兩個按鈕水平並排，刪除用自訂 Modal 確認

## 首頁版面結構
1. 頂部：帳單月份選擇器 ＋ 登出按鈕
2. KPI 三張卡片（世鴻藍、慧鳳紅、共同綠）
3. 三欄等寬區塊：
   - 左：帳單焦點總覽（總支出、較上期增減、交易筆數）
   - 中：支付工具統計（六項）
   - 右：消費行為統計（最高單筆、最高消費銀行、最多消費銀行）
4. 帳單匯入進度（五張卡片平均分配寬度）
5. 底部三張快捷卡片（帳單匯入、分類規則管理、對帳單預覽）
